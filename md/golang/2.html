认识函数 #

Go 不是一门纯函数式的编程语言, 但是函数在 Go 中是 "第一公民"

  • 函数是一种类型, 函数类型变量可以像其他类型变量一样使用, 可以作为其他函数的参数或返回值, 也可以直接调用执行
  • 函数支持多值返回
  • 支持闭包
  • 函数支持可变参数

基本概念 #

函数定义 #

go
func funcName(param-lsit)(result-list) {
  function-body
}
  • func: 关键字
  • funcName: 函数名
  • param-list: 参数列表
  • result-list: 返回列表
  • function-body: 函数主体

函数的特点 #

  1. 函数可以没有输入参数, 也可以没有返回值

  2. 多个相邻的相同类型的参数可以使用简写模式

    go
    func add(a, b int) int { // a int, b int 简写 a,b int
  return a + b
}

  3. 支持有名的返回值

    go
    package main

func add(a, b int) (sum int) {
	sum = a + b
	return
}
func main() {
	a := add(1, 2)
	println(a) // 3
}

  4. 不支持默认值传参

  5. 不支持函数重载

  6. 不支持函数嵌套, 严格来说是不支持命名函数嵌套, 但支持嵌套匿名函数

多值返回 #

go
package main

func swap(a, b int) (int, int) {
	return b, a
}
func main() {
	a, b := swap(1, 2)
	println(a) // 2
	println(b) // 1
}

实参到形参的传递 #

Go 函数实参到新参的传递永远是值拷贝

不定参数 #

  1. 所有的不定参数类型必须是相同的
  2. 不定参数必须是函数的最后一个参数
  3. 不定参数名在函数体内相当于切片, 对切片的操作童谣适合对不定参数的操作
  4. 切片可以作为参数传递给不定参数, 切片名要加上"..."
  5. 形参为不定参数的函数和形参为切片的函数类型不相同
go
package main

import "fmt"

func sum(arr ...int) (sum int) {
	for _, v := range arr {
		sum += v
	}
	return
}
func main() {
	slice := []int{1, 2, 3, 4, 5}
	// 数组不可以作为实参传递给不定参数的函数
	// array := [...]int{1, 2, 3, 4, 5}
	c := sum(slice...)
	fmt.Printf("%T\n%v", sum, c)
}

函数签名和匿名函数 #

函数签名 #

  • 函数类型又叫函数签名, 一个函数的类型就是函数定义首行去掉函数名, 参数名和 {}
  • 函数类型和 map, slice, chan 一样, 实际函数类型变量和函数名都可以当做指针变量, 未初始化的函数类型的变量的默认值是 nil
  • 可以直接使用函数名调用函数, 也可以直接赋值给函数类型变量, 后续通过该变量来调用该函数

匿名函数 #

  • 匿名函数可以直接赋值给函数变量
  • 可以当作实参, 也可以作为返回值, 还可以直接被调用

defer #

先注册后执行

  • defer 后面必须是函数或方法的调用, 不能是语句

  • defer 函数的实参在注册时通过值拷贝传递进去, 所以后面 a++ 并不会影响 defer 语句最后的输出结果

    go
    package main

func f() int {
	a := 0
	defer func(i int) {
		println("defer i=", i)
	}(a)
	a++
	return a
}

func main() {
	f() // defer i= 0
}

  • 主动调用 os.Exit(int) 退出进程时, defer 将不再被执行(即使 defer 已经提前注册)

    go
    package main

import "os"

func main() {
	defer func() {
		println("defer")
	}()
	println("func body")
	os.Exit(1)
}

  • defer 的好处就是可以在一定程度上资源泄露

    go
    func CopyFile(dst, path string) (w int64, err error) {
	src, err := os.Open(path)
	if err != nil {
		return
	}
	defer src.Close()

	dsts, err := os.Create(dst)
	if err != nil {
		return
	}
	defer dsts.Close()

	w, err = io.Copy(dsts, src)
	return
}

闭包 #

概念 #

闭包由函数及其相关引用环境组合而成的实体, 一般通过在匿名函数中引用外部函数的局部变量或包全局变量构成闭包 = 函数 + 引用环境

  1. 多次调用该函数, 返回的多个闭包所引用的外部变量是多个副本, 原因是每次调用函数都会局部变量分配内存
  2. 用一个闭包函数多次, 如果该闭包修改了其引用的外部变量, 则每一次调用该闭包对该外部变量都有影响, 因为闭包函数共享外部引用
go
package main

func fa(a int) func(i int) int {
	return func(i int) int {
		println(&a, a)
		a = a + i
		return a
	}
}

func main() {
	f := fa(1)
	g := fa(1)
	println(f(1)) // 2
  // 多次调用 f 引用的是同一个副本
	println(f(1)) // 3
	println(g(1)) // 2
	println(g(1)) // 3
}
  1. 使用闭包是为了减少全局变量, 所以闭包引用全局变量不是好的编程方式
  2. 同一个函数返回的多个闭包共享该函数的局部变量
go
package main

func fa(base int) (func(int) int, func(int) int) {
	println(&base, base)
	add := func(i int) int {
		base += i
		println(&base, base)
		return base
	}
	sub := func(i int) int {
		base -= i
		println(&base, base)
		return base
	}
	return add, sub
}

func main() {
	// f, g 闭包引用的base 是同一个
	f, g := fa(0)
	// s, k 闭包引用的base 是同一个
	s, k := fa(0)
	// f, g 和 s, k引用不同的闭包变量
	println(f(1), g(2))
	println(s(1), k(2))

	/*
		运行结果
		0x14000068f10 0
		0x14000068f08 0
		0x14000068f10 1
		0x14000068f10 -1
		1 -1
		0x14000068f08 1
		0x14000068f08 -1
		1 -1
	*/
}

价值 #

  • 减少全局变量, 但是不够直接, 不够清晰, 除非是非常有价值的地方, 一般不建议使用闭包
  • 对象是附有行为的数据, 而闭包是附有数据的行为
  • 闭包仅仅是锦上添花的东西, 不是必不缺少的

panic 和 recover #

概念 #

  • 引发 panic 有两种情况, 一种是程序主动调用 panic 函数, 另一种是程序产生运行时错误, 由运行时检测并抛出

  • 发送 panic 后, 程序会从调用panic的函数位置或发生panic的地方立即返回, 逐层向上执行defer语句,然后逐层打印函数调用堆栈, 直到被recover捕获或运行到最外层函数而退出

  • panic 不带可以在函数正常流程中抛出, 在 defer 逻辑里也可以再次调用 panic 或抛出 panic, defer 里面的 panic 能够被后执行的 defer 捕获

  • recover() 用来捕获 panic, 阻止 panic 继续向上传递

  • recover() 只有在 defer 后面的函数体内被直接调用才能捕获 panic 终止异常, 否则返回 nil, 异常继续向外传递

go
// 捕获失败, 未在函数体内被直接调用
defer recover()
defer fmt.Println(recover())
// 捕获失败, 函数嵌套两层, 未直接被调用
defer func() {
  func() {
    println("defer inner")
    recover
  }()
}
  • 可以有连续多个 panic 被抛出, 连续多个 panic 的场景只能出现在延迟调用里面, 否则不会出现多个 panic 被抛出的场景. 但只有最后一次panic能被捕获
go
package main

import "fmt"

func main() {
	defer func() {
		if err := recover(); err != nil {
			fmt.Println(err)
		}
	}()
	defer func() {
		panic("first defer panic")
	}()
	defer func() {
		panic("second defer panic")
	}()
	panic("main body panic")
}
/*
	打印结果
	first defer panic
*/
  • 包在 init 函数引发的 panic 只能在 init 函数中获取, 在 main 中无法被捕获, 原因是 init 函数先于 main 执行

使用场景 #

  1. 程序遇到无法正常执行下去的错误, 主动调用 panic 函数结束程序运行
  2. 在调试程序时, 通过主动调用 panic 实现快速退出, panic 打印出的堆栈能够更快地定位错误

为了保证程序的健壮性, 需要主动在程序的分支流程上使用recover()拦截运行时错误

底层实现 #

底层实现有两个方法

  1. 看语言编译器源码, 分析其对函数的各个特性的处理逻辑
  2. 反汇编, 将可执行程序反汇编出来

汇编基础 #

术语描述
SB (Static base poniter)静态基址寄存器, 它和全局符号一起表示全局变量的地址
FP (Frame poniter)栈帧寄存器, 该寄存器指向当前函数调用栈帧的栈底位置
PC (Program counter)程序计数器, 存放下一条指令的执行地址, 很少直接操作该寄存器
SP (Stack pointer)栈顶寄存器, 一般在函数调用前主调函数设置 SP 的值对栈空间进行分配或回收
