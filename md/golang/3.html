简介 #

Go 语言的类型系统可以分为命名类型, 非命名类型, 底层类型, 动态类型, 静态类型 等

命名类型和非命名类型 #

  • 命名类型(Named Type): 类型可以通过标识符来表示
  • 非命名类型(Unanmed Type): 一个类型由预声明类型, 关键字和操作符组成
go
package main

import "fmt"

// 使用type声明的是命名类型
type Person struct {
	name string
	age  int
}

func main() {
	// 使用 struct 字面量声明的是未命名类型
	a := struct {
		name string
		age  int
	}{
		name: "tjl",
		age:  25,
	}
	b := Person{
		name: "yjl",
		age:  23,
	}
	fmt.Printf("%T\n", a) // struct {name string; age int}
	fmt.Printf("%v\n", a)
	fmt.Printf("%T\n", b) // main.Person
	fmt.Printf("%v\n", b)
}
  1. 未命名类型和类型字面量是等价的
  2. 预声明类型都属于命名类型

底层类型 #

  1. 预声明类型(Pre-declared types)和类型字面量(type literals)的底层类型是他们自身
  2. 自定义类型中的 newtype 的底层类型是逐层递归向下查找直到查到 oldtype 是预声明类型或类型字面量位置

类型相同和类型赋值 #

类型相同 #

  1. 两个命名类型相同的条件是两个类型声明的语句完全相同
  2. 命名类型和未命名类型永远不相同
  3. 两个未命名类型相同的条件是他们类型声明字面量的结构相同, 并且内部元素的类型相同
  4. 通过类型别名语句声明的两个类型相同

类型赋值 #

需满足的条件如下:

  1. T1 和 T2 的类型相同
  2. T1 和 T2 具有相同的底层类型, 并且 T1 和 T2 里面只有有一个是未命名类型
  3. T2 是接口类型, T1 是具体类型, T1 的方法集是 T2 的方法集的超集
  4. T1 和 T2 都是通道类型, 它们拥有相同的元素类型, 并且 T1 和 T2 中至少有一个是未命名类型
  5. a 是预声明标识符 nil , T2 是指针, 函数字面量, 切片, 字典, 通道, 接口类型中的一个
  6. a 是一个字面常量值, 可以用来表示类型 T 的值

类型强制转换 #

非常量类型的变量 x 可以强制转化并传递给类型 T, 需要满足如下任一条件:

  1. x 可以直接赋值给 T 类型变量
  2. x 的类型和 T 具有相同的底层类型
  3. x 的类型和 T 都是未命名的指针类型, 并且指针指向的类型具有相同的底层类型
  4. x 的类型和 T 都是整型,或者都是浮点型
  5. x 的类型和 T 都是复数类型
  6. x 是整数值 []byte 类型的值, T 是 string 类型
  7. x 是一个字符串, T 是 []byte 或 []rune

注意:

  1. 数值类型和 string 类型之前相互转换可能造成值部分丢失. string和数字之间的转换可以使用标准库strconv
  2. Go 语言没有语言机制, 支持指针的 interger 之间的直接转换, 可以使用标准库中的unsafe包进行处理
go
package main

import "fmt"

type Map map[string]string

func (m Map) Print() {
	for _, value := range m {
		fmt.Println(value)
	}
}

type iMap Map

func (m iMap) Print() {
	for _, value := range m {
		fmt.Println(value)
	}
}

func main() {
	mp := make(map[string]string, 10)
	mp["hi"] = "tjl"
	var ma Map = mp
	// 会报错, 虽然底层类型相同, 但是其中一个并不是未命名类型
	// var im iMap = ma
	// 强制类型转换
	var im iMap = iMap(ma)
	im.Print()
	ma.Print()
}

初始化 #

  1. 按照字段顺序进行初始化(不推荐)
  2. 制定字段名进行初始化(推荐)
  3. 使用 new 创建内置函数, 字段默认初始化为其类型的零值,返回值是指向结构的指针 (不常用)
  4. 一次初始化一个字段(不常用)
  5. 使用构造函数进行初始化(推荐)
go
package main

import "fmt"

type Person struct {
	name string
	age  int
}

func NewPerson(name string, age int) *Person {
	return &Person{
		name: name,
		age:  age,
	}
}

func main() {
	// 第一种
	a := Person{"涂江林", 24}
	fmt.Printf("%v\n", a)
	// 第二种
	b := Person{
		name: "涂江林",
		age:  24,
	}
	fmt.Printf("%v\n", b)
	// 第三种(返回值为结构的指针)
	c := new(Person)
	c.name = "涂江林"
	c.age = 24
	fmt.Printf("%v\n", *c)
	// 第四种
	d := Person{}
	d.name = "涂江林"
	d.age = 24
	fmt.Printf("%v\n", d)
	// 第五种(返回值为结构的指针)
	e := NewPerson("涂江林", 24)
	fmt.Printf("%v", *e)
}

特点 #

  1. 结构的字段可以是任意类型, 基本类型, 接口类型, 指针类型, 函数类型都可以作为 struct 的字段. 结构字段的类型名必须唯一

  2. struct 字段类型可以是普通类型, 也可以是指针

  3. 结构支持内嵌自身指针, 这也是实现树形和链表等复杂数据结构的基础

匿名字段 #

  1. 如果字段只给出了字段类型, 没有给出字段名, 则称这样的字段为"匿名字段".
  2. 被匿名嵌入的字段名默认必须是命名类型或命名类型的指针, 类型字面量不能作为匿名字段使用
  3. 匿名字段的字段名默认就是类型名, 如果匿名字段是指针类型, 则默认的字段名就是指针指向的类型名

方法 #

命名类型定义方法的语法格式如下:

go
// 类型方法接收者是值类型
func (t TypeName) MethodName(ParamList) (ReturnList) {

}

// 类型方法接收者是指针
func (t *TypeName) MethodName(ParamList) (ReturnList) {

}
  • t: 是接收者, 可以自由指定名称
  • TypeName: 为命名类型的类型名
  • MethodName: 为方法名, 是一个自定义标识符
  • ParamList: 形参列表
  • ReturnList: 返回值列表
go
package main

import "fmt"

type SliceInt []int

func (s SliceInt) Sum() int {
	sum := 0
	for _, i := range s {
		sum += i
	}
	return sum
}

func SliceInt_Sum(s SliceInt) int {
	sum := 0
	for _, i := range s {
		sum += i
	}
	return sum
}

func main() {
	var a SliceInt = []int{1, 2, 3, 4}
	fmt.Println(a.Sum())
	fmt.Println(SliceInt_Sum(a))
}

特点 #

  1. 可以为命名类型增加方法(除了接口), 非命名类型不能自定义方法

不能为[]int类型增加方法, 因为[]int是非命名类型. 命名接口类型本身就是一个方法的签名集合,所以不能为其增加具体的实现方法

  1. 为类型增加方法有一个限制, 就是方法的定义必须和类型的定义在同一个包内

不能再为 int bool 等预声明类型增加方法, 因为它们是命名类型

  1. 方法的命名空间的可见性和变量一样, 大写开头的方法可以在包外被访问, 否则只能在包内可见
  2. 使用 type 定义的自定义类型是一个新类型, 新类型不能调用原有类型的方法, 但是底层类型支持的运算可以被新类型继承

方法调用 #

一般调用 #

go
TypeInstanceName.MethodName(ParamList)
  • TypeInstanceName: 类型实例名或指向实例的指针变量名
  • MethodName: 类型方法名
  • ParamList: 方法实参

方法值 #

x.T 是一个函数的类型变量, 可以复制给其他变量, 并像普通函数名一样使用

方法表达式 #

go
package main

import "fmt"

type T struct {
	a int
}

func (t T) Get() int {
	fmt.Println(t)
	return t.a
}

func (t *T) Set(i int) {
	fmt.Println(&t)
	t.a = i
}

func main() {
	t := T{
		a: 1,
	}
	// 如下方法表达式调用是等价的
	// t := T{}
	// (*T).Set(&t, 2)
	// 普通方法调用
	t.Get()
	// 方法表达式调用
	(T).Get(t)

	f1 := T.Get
	f1(t)
}

方法集 #

  1. 如果接收者是值类型, 则传递的是值的副本(个人理解前端的深拷贝,不会影响原值)
  2. 如果接收者是指针类型, 则传递的是指针的副本(浅拷贝会影响原值)
  3. T 类型的方法集是 S
  4. *T 类型的方法集是 S 和*S
go
package main

import "fmt"

type Int int

func (a Int) Max(b Int) Int {
	if a >= b {
		return a
	} else {
		return b
	}
}

func (i *Int) Set(a Int) {
	*i = a
}

func (i Int) Print() {
	fmt.Printf("%d\n", i)
}

func main() {
	var a Int = 10

	a.Set(30)
	fmt.Printf("%v\n", a.Max(40)) // 40
	a.Print()                     // 30
	(&a).Set(100)
	(&a).Print() // 100
}

值调用和表达式调用的方法集 #

  1. 通过类型字面量显式地进行值调用和表达式调用, 可以看到这种情况下编译器不会做自动转换, 会进行严格的方法集检查
go
package main

type Data struct{}

func (Data) TestValue() {}

func (*Data) TestPointer() {}

func main() {
  // Data 里只有 TestValue 方法
  // *Data 里有 TestValue 和 TestPointer 方法
	Data.TestValue(struct{}{})     // 表达式调用
	(Data)(struct{}{}).TestValue() // 值调用
	// 显式调用
	(*Data)(&struct{}{}).TestPointer()
	(*Data)(&struct{}{}).TestValue()
}
  1. 通过类型变量进行值调用和表达式调用, 在这种情况下值调用会进行自动转换
go
package main

type Data struct{}

func (Data) TestValue() {}

func (*Data) TestPointer() {}

func main() {
	var a Data
	a.TestPointer()
	a.TestValue()
}

内嵌字段的初始化和访问 #

  • 字段访问可以通过点操作符"."获取到, 字段可以嵌套很多层, 只要内嵌的字段是唯一的即可, 不需要全路径进行访问
  • 不同嵌套层次可以有相同的字段, 此时最好使用完全路径进行访问如果不使用完全路径访问会返回最近的字段值, 实际数据结构定义尽量避开相同的字段
go
package main

import "fmt"

type X struct {
	a int
}

type Y struct {
	X
	b int
}

type Z struct {
	Y
	c int
}

func main() {
	x := X{
		a: 1,
	}
	y := Y{
		X: x,
		b: 2,
	}
	z := Z{
		Y: y,
		c: 3,
	}

	fmt.Printf("%v\n", z)
	fmt.Println(z.a, z.X.a, z.Y.X.a) // 1 1 1
	z = Z{}
	z.a = 2
	fmt.Println(z.a, z.X.a, z.Y.X.a) // 2 2 2
}

组合的方法集 #

组合机构的方法集有如下规则

  1. 若类型 S 包含匿名字段 T, 则 S 的方法集包含 T 的方法集
  2. 若类型 S 包含匿名字段*T, 则 S 的方法集包含 T 和*T的方法集
  3. 不管类型 S 中嵌入的你命字段是 T 还是*T, *S方法集总是包含 T 和*T方法集
go
package main

type X struct {
	a int
}

type Y struct {
	X
}

type Z struct {
	*X
}

func (x X) Get() int {
	return x.a
}

func (x *X) Set(i int) {
	x.a = i
}

func main() {
	x := X{
		a: 1,
	}

	y := Y{
		X: x,
	}
	println(y.Get())
	y.Set(2)
	println(y.Get())
	/*
		方法表达式调用
		由于 type Y 方法集合并没有 Set 方法, 所以 Y.Set无效
		Y 内嵌字段 X, 所以 *Y 的方法集是Set Get
	*/
	(*Y).Set(&y, 3)
	println(y.Get())
	z := Z{
		X: &x,
	}
	z.Set(4)
	println(z.Get())
}

函数类型 #

函数声明 #

  1. 函数也是一种类型, 可以在函数字面量类型的基础上定义一种命名函数类型
  2. 有名函数和匿名函数的函数标签和命名函数类型的底层类型相同,他们之间可以进行类型转换
  3. 可以为有名函数类型添加方法, 这种为一个函数类型添加方法的技法非常有价值, 可以方便地为一个函数增加"拦截"或"过滤"等额外功能
  4. 为有名函数类型添加方法, 使其与接口打通关系, 使用接口的地方可以传递函数类型的变量, 这为函数到接口的转换开启了大门
go
package main

import "fmt"

func add(a, b int) int {
	return a + b
}

type ADD func(int, int) int

func main() {
	var g ADD = add
	fmt.Printf("%T\n%d", g, g(1, 2))
}
