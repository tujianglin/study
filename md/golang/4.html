简介 #

  • 接口是一个编程规约, 也是一组方法签名的集合

  • 接口是没有具体实现逻辑的, 也不能定义字段

  • 实例: 和面向对象编程中的"对象"的概念相似

  • 变量: 变量的值和变量的类型

  • 空接口: interface{ }, 任意类型都被认为实现了空接口, 任意类型的实例都可以赋值或传递给空接口, 包括非命名类型的实例

概念 #

声明 #

  • 接口字面量类型的声明语法 (使用场景很少, 一般只有空接口类型变量的声明才会使用)
go
interface { }
  • 接口命名类型使用 type 关键字声明
go
type InterfaceName interface { }

方法声明 #

方法名 + 方法签名

go
package main

type Reader interface {
	Read(p []byte) (n int, err error)
}

type Writer interface {
	Write(p []byte) (n int, err error)
}

type ReadWriter interface {
	Reader
	Writer
}

声明新接口类型的特点 #

  1. 接口的命名一般以 "er" 结尾
  2. 接口定义的内部方法声明不需要 func 引导
  3. 在接口定义中, 只有方法声明没有方法实现

初始化 #

  1. 直接调用未初始化的接口变量的方法会产生panic
go
package main

type Printer interface {
	Print()
}

type S struct{}

func (s S) Print() {
	println("print")
}

func main() {
	var a Printer = S{}
	a.Print()
	// 以下代码没有初始化接口, 所以会产生panic
	var b Printer
	b.Print()
}

动态类型和静态类型 #

动态类型 #

  1. 接口绑定的具体实例的类型称为接口的动态类型
  2. 接口可以绑定不同类型的实例, 所以接口的动态类型是会随着其绑定的不同类型实例而发生变化的

静态类型 #

  1. 接口被定义时, 其类型就已经被确定, 这个类型叫接口的静态类型

运算 #

  1. 接口是一个抽象的类型, 接口像一层胶水, 可以灵活的解耦软件的每一个层次, 基于接口编程是 Go 语言编程的思想
  2. Go 语言提供两种语法结构来支持这两种需求, 分别是类型断言和接口类型查询
  3. 类型查询和类型断言具有相同的语义, 只是语法不同
  4. 类型查询使用 case 语句一次判断多个类型

类型断言 #

直接赋值 #

go
package main

import "fmt"

type Inter interface {
	Ping()
	Pang()
}

type St struct {
	Name string
}

func (St) Ping() {
	println("ping")
}

func (*St) Pang() {
	println("pang")
}

type Anter interface {
	Inter
	String()
}

func main() {
	// var st St
	// st.Name = "111"
	st := &St{"111"}
	st.Ping()
	var i interface{} = st
	// 由于i 没有实现 string 方法
	// o := i.(Anter)
	// o.String()
	// 获取到接口内的类型
	p := i.(Inter)
	p.Pang()
	fmt.Printf("%v", i)

}

comma, ok 表达式 #

go
if o, ok := i.(TypeName); ok {

}
  1. TypeName 是具体类型名
go
func main() {
	st := &St{"111"}
	var i interface{} = st

	if o, ok := i.(Inter); ok {
		o.Pang()
	}
}

语法格式 #

switch v := i.(type) {
		case type1:
  		xxx
		case type2:
  		xxx
		default:
  		xxx
	}

语义分析 #

  1. i 必须是接口类型
  2. case 语句后面可以跟非接口类型名, 也可以跟接口类型名

优点和使用形式 #

优点 #

  1. 解耦: 复杂系统进行垂直和水平的分割是常用的设计手段, 在层与层之间使用接口进行抽象和解耦是一种好的编程策略
  2. 实现泛型: 由于现阶段 Go 语言不支持泛型, 使用空接口作为函数或方法参数能够用在需要泛型的场景中

使用形式 #

  1. 作为结构内嵌字段
  2. 作为函数或方法的形参
  3. 作为函数或方法的返回值
  4. 作为其他接口定义的嵌入字段

空接口 #

  1. 没有任何方法的接口, 我们称之为空接口

  2. 空接口是弥补泛型的一种手段

  3. 空接口不是真的为空, 接口有类型和值两个概念
